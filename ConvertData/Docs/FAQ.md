# FAQ — Часто задаваемые вопросы

## 📌 Общие вопросы

### Q1: Что делает ConvertData?

**A:** ConvertData конвертирует данные о балочных узловых соединениях из Excel (.xls/.xlsx) в структурированный JSON формат. Приложение автоматически:
- Читает данные из многолистовых Excel файлов
- Применяет справочник геометрических характеристик профилей
- Обогащает неполные записи
- Устраняет дубликаты CONNECTION_CODE
- Экспортирует справочники

---

### Q2: Для кого предназначен проект?

**A:** Для инженеров-конструкторов и разработчиков систем расчёта строительных конструкций, которым нужно преобразовать табличные данные о соединениях в формат JSON для дальнейшей обработки.

---

### Q3: Какие версии .NET поддерживаются?

**A:** Проект нацелен на **.NET 10.0**. Для работы требуется установленная .NET 10 Runtime.

---

## 📂 Работа с файлами

### Q4: Какие форматы Excel поддерживаются?

**A:** Поддерживаются оба формата:
- **.xls** (старый формат, требует Microsoft Excel для конвертации)
- **.xlsx** (современный формат, работает без Excel через EPPlus)

---

### Q5: Нужен ли Microsoft Excel для работы?

**A:** 
- **Для .xlsx файлов:** НЕТ, используется библиотека EPPlus
- **Для .xls файлов:** ДА, требуется Excel для автоматической конвертации .xls → .xlsx

**Рекомендация:** Конвертируйте .xls в .xlsx заранее, чтобы не зависеть от Excel.

---

### Q6: Как ConvertData обрабатывает несколько листов в Excel?

**A:** ConvertData ищет и объединяет данные из следующих листов:

1. **Main** (обязательный) — основные данные
2. **geometry** (опционально) — геометрия пластин, фланцев, рёбер
3. **bolts** (опционально) — параметры болтов
4. **weld** (опционально) — параметры сварных швов

Данные из дополнительных листов автоматически объединяются с основным по `CONNECTION_CODE`.

---

### Q7: Что делать, если в Excel файле нет колонки CONNECTION_CODE?

**A:** ConvertData ищет колонку по следующим вариантам имён (без учёта регистра):
- `CONNECTION_CODE`
- `Code`
- `Код`

Если ни одна не найдена, будет ошибка. Переименуйте колонку в один из этих вариантов.

## 📊 Данные и форматы

### Q12: В каком формате записываются числа в JSON?

**A:** Числа записываются в **инвариантном формате** (точка как десятичный разделитель):

```json
{
  "Beam_H": 200.5,
  "Beam_B": 100.75
}
```

---

### Q13: Поддерживаются ли русские десятичные числа (с запятой)?

**A:** Да! NumericParser поддерживает оба формата:

**Входные данные в Excel:**
```
3,14  → 3.14
3.14  → 3.14
1 234,56 → 1234.56
```

---

### Q14: Что такое "обогащение неполных записей"?

**A:** Если несколько записей имеют одинаковый `CONNECTION_CODE`, но разную полноту данных:

**До обогащения:**
```json
[
  {
    "CONNECTION_CODE": "M-3",
    "Geometry": { ... },  // Полностью заполнено
    "Bolts": { ... },
    "InternalForces": { "Nt": 100 }
  },
  {
    "CONNECTION_CODE": "M-3",
    "Geometry": {},       // Пусто!
    "Bolts": {},          // Пусто!
    "InternalForces": { "Nt": 200 }
  }
]
```

**После обогащения:**
```json
[
  { ... },  // Без изменений
  {
    "CONNECTION_CODE": "M-3",
    "Geometry": { ... },  // ← СКОПИРОВАНО из первой записи
    "Bolts": { ... },     // ← СКОПИРОВАНО
    "InternalForces": { "Nt": 200 }  // Осталось своё значение
  }
]
```

**Копируются:** `Geometry`, `Bolts`, `Welds`, `TableBrand`  
**Остаются уникальными:** `InternalForces`, `Coefficients`

---

### Q15: Как работает устранение дубликатов CONNECTION_CODE?

**A:** ConvertData находит дубликаты и переименовывает их:

**Входные данные:**
```
M-3 (оригинал)
M-3 (дубликат)
M-3 (дубликат)
M-5 (оригинал)
M-5 (дубликат)
```

**Результат:**
```
M-3 (оставлен)
M-6 (было M-3)
M-7 (было M-3)
M-5 (оставлен)
M-8 (было M-5)
```

**Отчёт** сохраняется в `CONNECTION_CODE_replacements.txt`:
```
M-3 => M-6
M-3 => M-7
M-5 => M-8
```

## 🗂️ Справочник профилей

### Q16: Где взять файл ProfileBeam.xls?

**A:** Этот файл должен быть предоставлен вашей организацией или создан вручную. Формат:

**Лист ProfileI (двутавры):**

| Profile | H | B | s | t | A | P | Iz | Iy | Wz | Wy | Sz | Sy | iz | iy |
|---------|---|---|---|---|---|---|----|----|----|----|----|----|----|----|
| 20Б1 | 200 | 100 | 5.6 | 8.5 | 30.6 | 24.0 | 1840 | 155 | 184 | 24.2 | 104 | 15.6 | 7.75 | 2.25 |

**Листы:** ProfileI, ProfileC (швеллеры), ProfileL (уголки)

---

### Q17: Что делать, если справочника профилей нет?

**A:** Приложение будет работать, но геометрические характеристики балок и колонн останутся незаполненными. Данные из Excel (если они есть) будут использованы, но автоматического обогащения не произойдёт.

---

### Q18: Можно ли добавить новые типы профилей?

**A:** Да! В `ProfileExcelToJsonExporter.cs` измените словарь:

```csharp
private static readonly Dictionary<string, string> FileCategoryMap = new()
{
    ["ProfileI.xlsx"] = "Двутавр",
    ["ProfileC.xlsx"] = "Швеллер",
    ["ProfileL.xlsx"] = "Уголок",
    ["ProfileT.xlsx"] = "Тавр",  // ← Добавьте новый тип
};
```

Создайте файл `ProfileT.xlsx` в `EXCEL_Profile/` с соответствующими колонками.

## 🐛 Ошибки и устранение неполадок

### Q19: Ошибка "Cannot find CODE column"

**Причина:** В Excel нет колонки с кодом соединения.

**Решение:**
1. Откройте Excel файл
2. Убедитесь, что есть колонка `CONNECTION_CODE`, `Code` или `Код`
3. Проверьте, что заголовок в первой строке данных

---

### Q20: Ошибка "File is being used by another process"

**Причина:** Excel файл открыт в другой программе.

**Решение:**
1. Закройте все программы, использующие файл
2. Запустите ConvertData снова

---

### Q21: Ошибка COM Interop при конвертации .xls

**Причина:** Microsoft Excel не установлен или недоступен.

**Решение:**
- **Вариант 1:** Установите Microsoft Excel
- **Вариант 2:** Конвертируйте .xls в .xlsx вручную (Файл → Сохранить как → .xlsx)
- **Вариант 3:** Используйте только .xlsx файлы

---

### Q22: В JSON файле некоторые числа равны 0

**Причина:** 
1. В Excel ячейки пустые
2. Формат числа не распознан (например, текст вместо числа)
3. Ошибка парсинга

**Решение:**
1. Проверьте Excel: ячейки должны иметь числовой формат
2. Убедитесь, что используете `.` или `,` как десятичный разделитель
3. Удалите лишние символы (пробелы, невидимые символы)

---

### Q23: Некоторые записи не обогащаются

**Причина:** Система не нашла запись-шаблон с достаточной полнотой.

**Решение:**
1. Убедитесь, что хотя бы одна запись с данным `CONNECTION_CODE` имеет заполненные `Geometry`, `Bolts`, `Welds`
2. Проверьте отчёт в консоли: сколько записей было обогащено
3. Если шаблон не найден, система пропускает группу

## 🔐 Безопасность и лицензирование

### Q26: Какая лицензия у EPPlus?

**A:** EPPlus 7.x использует лицензию **Polyform Noncommercial**. ConvertData настроен на режим `NonCommercial`:

```csharp
ExcelPackage.LicenseContext = LicenseContext.NonCommercial;
```

Для коммерческого использования требуется покупка лицензии EPPlus.

---

### Q27: Безопасно ли запускать ConvertData?

**A:** Да, приложение:
- Не отправляет данные в интернет
- Не модифицирует исходные Excel файлы
- Создаёт только новые JSON файлы
- Использует COM Interop только для чтения .xls файлов

## 🛠️ Разработка и расширение

### Q30: Как добавить новый формат ввода (например, CSV)?

**A:** 

1. Реализуйте интерфейс `IRowReader`:

```csharp
public class CsvRowReader : IRowReader
{
    public List<Row> Read(string path)
    {
        // Логика чтения CSV
    }
}
```

2. Зарегистрируйте в `RowReaderFactory`:

```csharp
public IRowReader Create(string path)
{
    var ext = Path.GetExtension(path).ToLowerInvariant();
    
    if (ext == ".csv")
        return new CsvRowReader();
    
    if (ext == ".xls" || ext == ".xlsx")
        return new EpplusRowReader();
    
    throw new NotSupportedException($"Format {ext} not supported");
}
```

См. подробнее в [Architecture.md](Architecture.md) → "Точки расширения".

---

### Q31: Как добавить новый формат вывода (например, XML)?

**A:**

1. Реализуйте интерфейс `IRowWriter`:

```csharp
public class XmlRowWriter : IRowWriter
{
    public void Write(List<Row> rows, string outputPath)
{
        // Логика записи XML
    }
}
```

2. Используйте в `ConvertApp`:

```csharp
var writer = new XmlRowWriter();
writer.Write(rows, "output.xml");
```

---

### Q32: Где хранятся временные файлы?

**A:** Временные .xlsx файлы (после конвертации .xls) создаются в:

```
C:\Users\<Username>\AppData\Local\Temp\
```

Формат имени: `<OriginalName>_converted_<GUID>.xlsx`

Файлы автоматически удаляются после обработки.

### Q41: Как проверить качество конвертации?

**A:**

1. **Проверьте отчёты:**
   - `CONNECTION_CODE_duplicates.txt` — дубликаты
   - `CONNECTION_CODE_replacements.txt` — замены

2. **Откройте JSON файлы:**
   - Убедитесь, что все поля заполнены
   - Проверьте несколько записей вручную

3. **Сравните с Excel:**
   - Выберите случайную запись
   - Сверьте значения полей

4. **Используйте JSON валидатор:**
   - https://jsonlint.com
   - Проверьте синтаксис JSON
