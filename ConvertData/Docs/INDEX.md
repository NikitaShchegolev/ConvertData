## 🔄 Обновление документации

### Правила обновления

1. **При изменении архитектуры:**
   - Обновить `Architecture.md`
   - Обновить `UML.md` (диаграмма классов)
   - Обновить `Mermaid.md` (архитектура слоёв)

2. **При добавлении нового API:**
   - Добавить в `API.md`
   - Обновить примеры в `README.md`
   - Обновить диаграммы в `UML.md` и `Mermaid.md`

3. **При изменении потока данных:**
   - Обновить `DataFlow.md`
   - Обновить `Mermaid.md` (поток данных)
   - Проверить соответствие `README.md`

4. **При изменении формата данных:**
   - Обновить `README.md` (раздел "Формат данных")
   - Обновить `API.md` (описание Row)
   - Обновить `Mermaid.md` (структура Row)

### Версионирование документации

Документация следует версии проекта:
- **Мажорные изменения** (1.0 → 2.0): полный аудит всех документов
- **Минорные изменения** (1.0 → 1.1): обновление затронутых разделов
- **Патчи** (1.0.0 → 1.0.1): исправление опечаток, уточнения
